# Data-Science-Final-Project

## Project Overview:
This project analyzes the New York City Department of Health and Mental Hygiene (DOHMH) restaurant inspection data. The goal is to identify geographic hotspots for health violations and determine which factors, such as borough or cuisine type, correlate most strongly with inspection failures.

## Questions:
- Which cuisines consistently receive the highest inspection grades?
- Is there a correlation between a borough and its likelihood of failing an inspection?
- Which zip codes have a higher density of violations per restaurant than the city average?
- What are the most common health violations cited across the five boroughs?

## Repository Structure:
- Data Science Project.pdf: Project goals and dataset selection.
- milestonetwo.ipynb: Initial data exploration and statistics.
- milestonethree.ipynb: Data cleaning (handling missing values and invalid dates).
- milestonefour.ipynb: Final analysis with visualizations and insights.
- datasetlinks.pdf: PDF containing the links to the processed and initial dataset.
- NYC Restaurant Inspection Analysis.pdf: A one-page summary covering tools used, insights, and recommendations for the DOHMH.
- Data Science Final Presentation.pdf: The final presentation summarizing the project methodology and findings.
- README.md: This file, providing an overview and guide to the repository.

## Tools Used:
- Python: Main programming language.
- Pandas: Used for data cleaning, grouping, and calculating violation density.
- Matplotlib: Used to generate bar charts and horizontal frequency plots.
- Jupyter Notebook: Development environment used.
